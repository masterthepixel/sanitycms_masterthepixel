# Story: Image & Content Sanitization (MIG-009.02)

## User Story
As a developer, I want all `.mdx` files and static content to use local `/public/assets` images natively so that the UI is identical without relying on Sanity's CDN.

## Acceptance Criteria
- [ ] No `cdn.sanity.io` URLs exist in `content/pages/*.mdx`, `content/posts/*.mdx`, etc.
- [ ] All required images are firmly located in `/public/assets/*`.
- [ ] React UI components handle image cropping/resizing natively, bypassing Sanity's `?w=1200&fit=crop` CDN math.
- [ ] Page designs, spacing, and aspect ratios match the current live UI exactly.

## Technical Implementation Notes
- Use a `grep` or search to find `https://cdn.sanity.io` in the `content/` folder.
- Ensure any `feature.image` or `hero.bgImage` fields map to local relative paths (e.g., `/assets/hero-bg.jpg`).
- **Edge Case (Hotspot/Crop):** Sanity components might expect an image `hotspot` property object. Review `src/components/` blocks (like `hero` or `article-card`). If they use Sanity's `urlFor` or `imageUrlBuilder`, refactor them to use standard `next/image` with `layout="fill"` or `style={{ objectFit: 'cover' }}`.
