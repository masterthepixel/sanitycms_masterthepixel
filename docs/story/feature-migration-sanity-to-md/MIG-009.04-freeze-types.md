# Story: Freeze Types & Remove Dependencies (MIG-009.04)

## User Story
As a developer, I want to remove `@sanity` NPM packages while freezing our current TypeScript shapes as standard artifacts, so that our CI typing passes without carrying bulk dependencies.

## Acceptance Criteria
- [ ] `package.json` contains no `@sanity/*`, `next-sanity`, or `sanity` dependencies in `dependencies` or `devDependencies`.
- [ ] Codebase compiles successfully with `npx tsc --noEmit`.
- [ ] `sanity.types.ts` is retained but acts purely as a static TS definition file.
- [ ] Typegen scripts (e.g., `npx sanity typegen`) are removed from `package.json` scripts.

## Technical Implementation Notes
- Removing the packages is easy, but it will immediately break any `import { PortableText } from '@portabletext/react'` or `import { SanityDocument } from 'next-sanity'`.
- You will need to replace `PortableText` with standard MDX components if content relies on it. For pure types imported from `next-sanity` (like `SanityImageSource`), you will need to map these to standard local interfaces, or stub them inside `types/index.ts`.
- `sanity.types.ts` should remain untouched. Let it serve as the interface truth map for the shapes expected by your Page Builder components for now.
