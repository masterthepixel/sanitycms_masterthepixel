# Story: Decouple Build & Config from Sanity (MIG-009.01)

## User Story
As a developer, I want to remove Sanity dependencies from the Next.js build configuration so that Vercel builds do not crash when Sanity environment variables are removed.

## Acceptance Criteria
- [ ] `next.config.ts` no longer fetches redirects from Sanity.
- [ ] Redirects are loaded locally from `data/redirects.json` or hardcoded in `next.config.ts`.
- [ ] `cdn.sanity.io` is removed from `remotePatterns` / `images.domains` in `next.config.ts` (if all images are local).
- [ ] Build succeeds locally (`pnpm run build`) without requiring `NEXT_PUBLIC_SANITY_PROJECT_ID` or `SANITY_API_READ_TOKEN`.

## Technical Implementation Notes
- Look for the `redirects()` function in `next.config.ts`. Currently, it likely imports a Sanity client or fetch function to query custom redirects.
- Rewrite the `redirects()` function to import/read `data/redirects.json` directly.
- **Risk Mitigation:** Before deleting the Sanity fetch block, ensure `data/redirects.json` actually contains all existing redirects. You may need to run one final script to export them if they are missing.
