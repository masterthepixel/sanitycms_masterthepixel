# Story: Clean Up Studio, Scripts & Documentation (MIG-009.05)

## User Story
As a developer, I want to delete the embedded Sanity Studio and all related migration scripts while standardizing our documentation, so that the repository represents a clean, file-based purely Next.js system.

## Acceptance Criteria
- [ ] `studio/masterthepixel` is deleted entirely.
- [ ] `archive/studio` is deleted.
- [ ] `scripts/` directory is cleared of all Sanity migration scripts (e.g., `check-content.js`, `sanity-to-mdx.js`, `download-assets.js`, etc.).
- [ ] Remaining `.env.example` and CI deployment documentation mention no Sanity tokens.
- [ ] `README.md` is updated to define the new "Local MDX Authoring Flow" emphasizing that content is written in VS Code directly.

## Technical Implementation Notes
- The studio folder might have its own `package.json` or locks—removing the folder fully handles it, but check the root `pnpm-workspace.yaml` to ensure it's not looking for `studio/*`.
- For the `README.md` updates, include the process for adding a new blog post or page (e.g., "duplicate `content/posts/example.mdx`, change frontmatter, add image to `/public/assets`, and run `pnpm run dev` to preview locally").
- Verify that standard deployment to Vercel requires no manual intervention with these scripts or folders removed.
