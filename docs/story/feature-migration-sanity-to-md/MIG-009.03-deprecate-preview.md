# Story: Deprecate Live Preview & Draft Modes (MIG-009.03)

## User Story
As a developer, I want to remove Sanity's real-time live preview API endpoints and draft logic so that the site doesn't attempt to open streaming network connections to a disabled backend.

## Acceptance Criteria
- [ ] `src/app/(backend)/api/draft/` (or similar draft-mode routes) are deleted.
- [ ] `src/sanity/lib/live.ts` and related live querying wrappers are removed.
- [ ] The `VisualEditing` or `SanityLive` wrapper components are removed from `src/app/layout.tsx` (and any other entry points).
- [ ] UI components do not throw runtime frontend errors about missing API tokens or disabled preview endpoints.

## Technical Implementation Notes
- Delete the API routes governing the preview session. Next.js draft mode relies on cookies set by these routes.
- Thoroughly check `layout.tsx` and `page.tsx` for any conditionals checking `draftMode().isEnabled`. All data fetching should default entirely to standard `src/lib/content.ts` reads.
- Ensure the removal of live previews does not inadvertently break the standard `getPageBySlug` or `getAllPosts` rendering pipelines.
